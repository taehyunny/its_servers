## 🤝 개발 협업 가이드라인 (Rules)

1. 개발 환경: Server(Linux C++ / MariaDB), Clients(Qt, Python, MFC)
2. 네이밍 규칙: 통신 JSON 데이터 및 변수명은 카멜 케이스(camelCase)를 엄격히 준수합니다.
   - 올바른 예: `orderId`, `totalPrice`, `deliveryAddress` (첫 글자 소문자, 이어지는 단어 첫 글자 대문자)
   - 잘못된 예: `order_id` (스네이크), `OrderId` (파스칼)
3. Git 브랜치 및 병합: `main` 브랜치에 직접 푸시(Push)하지 않습니다. 각자의 기능 브랜치(`feature/login` 등)에서 작업 후, 병합 전 확인 부탁드립니다.
4. 통신 약속: `Global_protocol.h`의 헤더 구조체가 완벽히 일치하는지 확인 후 패킷 전송 테스트를 진행합니다.
5. 주석 필수: 서로 다른 언어와 프레임워크가 통신하는 프로젝트입니다. 
   본인이 짠 코드의 핵심 로직(특히 네트워크 송수신부)에는 반드시 상세한 주석을 달아주세요.

## Git 사용방법 

1. main (또는 master) 브랜치 직접 수정 금지
main 브랜치는 항상 실행 가능한, 에러가 없는 깨끗한 상태를 유지해야 합니다.
절대 main에서 직접 코드를 짜거나 커밋하지 마세요. 새로운 작업은 무조건 본인의 브랜치를 새로 생성(git checkout -b 브랜치명)해서 진행해야 합니다.

2. 브랜치 이름 규칙 (Naming Convention) 통일
브랜치 이름만 봐도 누가, 어떤 작업을 하고 있는지 단번에 알 수 있어야 합니다.
일반적으로 작업종류/작업내용 형태를 씁니다.
새 기능 추가: feature/login-ui
버그 수정: fix/db-connection
문서 수정: docs/readme-update
팀원들과 어떤 접두사를 쓸지 사전에 꼭 합의하세요.

3. 작업 단위는 최대한 작게, 커밋은 자주
하나의 브랜치에서 며칠 치 작업을 몽땅 몰아서 하면, 나중에 원본에 합칠 때 어마어마한 충돌이 발생할 확률이 높습니다.
'로그인 UI 만들기', '로그인 로직 구현하기' 처럼 하나의 기능 단위로 브랜치를 파고, 완료되면 즉시 메인에 합치는 것이 좋습니다.

4. 내 작업을 합치기 전, 항상 최신 코드 Pull 받기
내가 작업하는 동안 다른 팀원이 이미 코드를 완성해서 main 브랜치를 업데이트했을 수 있습니다.
내 코드를 메인에 반영하기(Merge) 전에는, 반드시 최신 상태의 main 코드를 내 브랜치로 당겨와서(Pull) 내 코드가 최신 버전과 충돌하지 않고 잘 돌아가는지 내 PC에서 먼저 테스트해야 합니다.

5. 충돌(Conflict)이 났을 때 혼자 임의로 덮어쓰지 않기
동일한 파일의 같은 줄을 두 명이 동시에 수정하면 병합할 때 충돌 메시지가 뜹니다.
이때 에러를 빨리 없애려고 '내 코드로 덮어쓰기'를 해버리면 열심히 짠 다른 팀원의 코드가 통째로 날아가 버립니다.
충돌이 발생한 부분은 반드시 해당 코드를 작성한 팀원과 직접 소통하며 어떤 코드를 남기고 지울지 상의해서 해결해야 합니다.

## 명령어 
1. 현재 작업 내용 저장하기 (Commit)
먼저 노란색 'M' 표시가 뜬 파일들의 수정 사항을 현재의 taehyun 브랜치에 확정 지어야 합니다.

git add . (수정된 모든 파일을 올릴 준비)

git commit -m "작업한 내용에 대한 설명 작성"  

2. 메인 브랜치로 이동하기 (Switch/Checkout)
작업물을 흡수할 바탕이 되는 main 브랜치로 넘어갑니다.

git switch main (과거 버전의 Git이라면 git checkout main)

3. 최신 메인 상태 동기화 (Pull) - 매우 중요
화면을 보면 origin/main이 이미 다른 커밋들로 앞서 있습니다. 내 로컬 PC의 main을 최신 상태로 맞춰주어야 합니다.

git pull origin main

4. 내 브랜치 병합하기 (Merge)
이제 최신 상태로 업데이트된 main 위에서 taehyun 브랜치의 작업물을 합칩니다.

git merge (체크아웃 된 이름)

5. 원격 저장소에 최종 반영하기 (Push)
내 PC에서 성공적으로 합쳐진 최종 결과를 서버에 올려 팀원들과 공유합니다.

git push origin main
